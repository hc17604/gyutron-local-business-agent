# GyuTron Local Agent 产品需求文档

## 1. 产品定位

GyuTron Local Agent 是一个面向跨境制造企业、跨境电商企业、工业品外贸公司的本地部署 AI 业务 Agent。它部署在客户本地电脑、服务器或内网环境中，默认保存客户数据、字段映射、业务规则、历史报告和报告模板到本地。

它不是普通数据看板，不是 SaaS 云平台，不是简单聊天机器人，也不是让客户自己搭建复杂工作流的平台。第一版目标是做一个开箱即用的本地老板日报 Agent：用户上传 Alibaba、Shopee、Amazon、ERP 等平台导出的 Excel/CSV，配置自己的模型 API Key 和业务规则，系统生成老板能看懂的经营日报。

## 2. MVP 边界

### 2.1 第一版必须做

1. 本地 Web 控制台。
2. Excel/CSV 上传。
3. 支持数据类型：询盘、订单、产品、库存、广告、客户、销售跟进。
4. 读取表头和前 20 行数据。
5. 自动猜测字段含义，例如客户名、国家、产品、金额、时间、状态、销售员。
6. 用户手动修正字段映射。
7. 字段映射、本地规则、历史报告保存到 SQLite。
8. 用户配置模型 API Key、Base URL、Model Name。
9. 第一版只支持 OpenAI-compatible API。
10. 用户添加自然语言业务规则。
11. 根据上传数据、字段映射、业务规则和模型配置生成老板日报。
12. 报告保存到本地，可查看历史报告。
13. 第一版只读数据，不自动发邮件，不自动修改平台数据，不自动执行高风险动作。

### 2.2 第一版明确不做

- 不做 SaaS 多租户。
- 不默认上传客户业务数据到 GyuTron 云端。
- 不接 Alibaba、Shopee、Amazon、TikTok Shop 等真实平台 API。
- 不做复杂权限系统。
- 不做工作流编排平台。
- 不自动发送邮件、企业微信、Slack 或 WhatsApp。
- 不自动修改订单、库存、广告、客户资料。
- 不在单元测试中依赖真实外部 LLM。

## 3. 目标用户

- 外贸工厂老板、总经理、业务负责人。
- 跨境电商公司老板、运营负责人。
- 工业品外贸公司销售主管、跟单主管。

核心诉求：

- 每天快速知道业务异常和机会。
- 不想把完整业务数据交给第三方 SaaS。
- 不想让团队维护复杂 BI 或自动化工作流。
- 希望 AI 能读懂业务表格，并给出可行动建议。

## 4. 核心用户流程

### 4.1 首次启动

1. 用户在本机或服务器启动 Docker Compose。
2. 打开本地 Web 控制台。
3. 系统显示本地运行状态和隐私提示。

### 4.2 配置模型

1. 用户进入模型设置。
2. 填写 Base URL、API Key、Model Name。
3. 系统保存到本地 SQLite。
4. 用户点击测试连接。

### 4.3 导入数据

1. 用户选择数据类型。
2. 上传 Excel/CSV。
3. 系统保存原始文件到 `data/uploads`。
4. 系统读取表头和前 20 行。
5. 系统自动猜测字段映射。
6. 用户手动修正并保存。

### 4.4 添加业务规则

用户可以添加自然语言规则，例如：

- 巴西客户优先级提高。
- 超过 24 小时未回复的询盘要预警。
- 毛利低于 20% 的订单标红。
- 工业相机和扫码枪是重点产品。

规则保存在本地 SQLite，生成报告时作为高优先级上下文。

### 4.5 生成老板日报

1. 用户选择已导入的数据集。
2. 系统读取字段映射和启用中的业务规则。
3. 系统先生成结构化数据摘要。
4. 系统调用用户配置的 OpenAI-compatible 模型。
5. 系统保存报告到 SQLite 和本地报告目录。
6. 用户查看历史报告。

## 5. 报告内容要求

老板日报必须包含：

1. 老板摘要。
2. 核心数据变化。
3. 异常提醒。
4. 高优先级客户。
5. 产品机会。
6. 国家/市场趋势。
7. 销售跟进任务。
8. 风险点。
9. 下一步建议。

报告原则：

- 使用中文。
- 面向老板和业务负责人，不面向程序员。
- 不编造数据摘要中没有的事实。
- 对数据不足的地方说明“不足以判断”。
- 明确体现用户业务规则。

## 6. 标准字段方向

MVP 只做足够生成老板日报的标准字段，不做复杂主数据系统。

常见字段包括：

- 客户：客户名、邮箱、电话、公司、国家、来源平台、客户等级、负责人。
- 询盘：询盘时间、产品、数量、消息、状态、最后回复时间。
- 订单：订单号、订单时间、客户、国家、产品、数量、金额、成本、毛利、状态。
- 产品：SKU、产品名、分类、售价、成本、库存、状态。
- 库存：SKU、仓库、库存数、可用数、安全库存、更新时间。
- 广告：日期、平台、活动、曝光、点击、花费、询盘、订单、销售额。
- 跟进：客户、负责人、跟进时间、下次跟进时间、状态、内容。

## 7. 验收标准

MVP 阶段验收：

- 本地可以启动前端和后端。
- `GET /health` 正常返回服务状态。
- SQLite 数据库初始化到 `data/db`。
- 用户可以在 Web 控制台看到后端健康状态。
- 项目目录符合 `apps/api`、`apps/web`、`docs`、`data` 的结构。
- 文档清楚描述产品边界、架构、路线图和长期开发规则。

后续功能验收：

- 可以上传 CSV/XLSX 并预览前 20 行。
- 可以保存和复用字段映射。
- 可以保存模型配置且不在日志打印 API Key。
- 可以保存业务规则。
- 可以用摘要调用模型生成老板日报。
- 可以查看历史报告。
